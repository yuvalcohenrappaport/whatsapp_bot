🧠 AI Pipeline Details

1:1 Reply Generation

Model: Gemini 2.5 Flash
System prompt: Name + relationship + custom instructions + style summary
Few-shot: 15 randomly sampled from 200 recent messages
Safety: Filters model thinking leakage (THINK/THOUGHT prefixes)
Auto-cap: 20 consecutive replies → switch to draft mode

Date Extraction

Schema: Zod — title, date (ISO 8601), confidence
Pre-filter: Skip messages without digits (save API calls)
Filter: High-confidence only (pessimistic approach)
Timezone: Asia/Jerusalem

Travel Search

Trigger: @bot mention or reply to travel result
Intent parsing: Zod schema — destination, dates, budget, preferences
Context: Last 2h of group messages (up to 20) for inference
Rate limit: 30s per group, reply chains capped at 500

Keyword Auto-Response

Matching: Case-insensitive substring or regex
Response types: Fixed text or Gemini AI-generated
Strategy: First-match-wins, non-terminal (continues pipeline)
Cooldown: Per-rule, ephemeral (resets on restart)

Weekly Digest

Trigger: node-cron per group (configurable day/hour)
Input: 7 days of messages + 14 days of calendar events
Output: Structured sections — Events, Tasks, Notes

Socket Lifecycle
Baileys socket can't be reused; new socket on each reconnect

Travel Result Map
In-memory reply chain tracking, capped at 500 entries
Rule Cooldowns

Auto-reply Counter
