Create `src/voice/tts.ts` exporting a single async function:

```typescript
export async function textToSpeech(
  text: string,
  logger: pino.Logger,
): Promise<Buffer>
```

The module has three internal stages. Implement all three in this one file:

**Stage 1 — ElevenLabs TTS API call:**
- Import `elevenLabsClient` from `'./client.js'`
- Import `{ config }` from `'../config.js'` (for ELEVENLABS_DEFAULT_VOICE_ID)
- Call `elevenLabsClient.textToSpeech.convert(config.ELEVENLABS_DEFAULT_VOICE_ID, { ... })` with:
  - `text` — the input parameter
  - `modelId: 'eleven_v3'` — CRITICAL: NOT eleven_multilingual_v2 (does not support Hebrew)
  - `outputFormat: 'mp3_44100_128'` — mp3 is reliable for ffmpeg auto-detection
- The result is a `ReadableStream<Uint8Array>` (Web Streams API, NOT Node.js Readable)

**Stage 2 — Collect ReadableStream to Buffer:**
- Create a private helper `streamToBuffer(stream: ReadableStream<Uint8Array>): Promise<Buffer>`
- Use the `getReader()` loop pattern:
  ```
  const reader = stream.getReader();
  const chunks: Buffer[] = [];
  while (true) {
    const { done, value } = await reader.read();
    if (done) break;
    chunks.push(Buffer.from(value));
  }
  return Buffer.concat(chunks);
  ```
- Log at debug level: `{ bytes: mp3Buffer.length }, 'TTS MP3 received'`

**Stage 3 — Transcode MP3 to OGG/Opus via ffmpeg:**
- Import `{ spawn }` from `'child_process'`
- Import `ffmpegPath` from `'ffmpeg-static'` (default import, works with esModuleInterop)
- Create a private helper `transcodeToOgg(mp3Buffer: Buffer, logger: pino.Logger): Promise<Buffer>`
- CRITICAL: Check `if (!ffmpegPath) throw new Error('ffmpeg-static binary not found')` before spawning
- Spawn ffmpeg with args: `['-i', 'pipe:0', '-c:a', 'libopus', '-b:a', '64k', '-f', 'ogg', 'pipe:1']`
- Use `{ stdio: ['pipe', 'pipe', 'pipe'] }` to get stdin/stdout/stderr
- Collect stdout chunks into Buffer[] for the OGG output
- Collect stderr chunks for error reporting
- Handle events:
  - `proc.stdout.on('data', ...)` — accumulate output chunks
  - `proc.stderr.on('data', ...)` — accumulate error output
  - `proc.on('close', (code) => ...)` — if code !== 0, reject with last 500 chars of stderr; if code === 0, resolve with concatenated output
  - `proc.on('error', reject)` — spawn failure
  - `proc.stdin.on('error', (err) => { if (err.code !== 'EPIPE') reject(err) })` — EPIPE is expected if ffmpeg closes stdin early; swallow it, the 'close' event will surface the real error
- Write input: `proc.stdin.end(mp3Buffer)`
- CRITICAL: Do NOT use `-f opus` as input format — it is OUTPUT-ONLY in ffmpeg-static 7.0.2. Use `-i pipe:0` without `-f` so ffmpeg auto-detects mp3.

**After transcode:**
- Log at info level: `{ bytes: oggBuffer.length }, 'TTS OGG ready'`
- Return the OGG Buffer

Do NOT add try/catch in the exported function — let the caller handle errors.
Use `.js` import extensions per project convention.
Type the `err` in EPIPE handler as `NodeJS.ErrnoException` for the `.code` property.
