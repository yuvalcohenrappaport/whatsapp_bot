### Phase 52 Рђћ Multimodal Intake

- [ ] **MM-01**: Images / PDFs / documents dropped into a `travelBotActive` group are parsed by `gemini-2.5-flash` vision against a structured `TripFactExtraction` schema (type, title, date, time, location, reservation_number, cost, confidence, notes). Extractions with `confidence >= 0.8` insert as `trip_decisions` with `origin='multimodal'`. Stickers and images <50KB are pre-filtered and skipped
- [ ] **MM-02**: When a multimodal extraction includes both `date` and `time`, the same suggest-then-confirm flow from v1.4 (`createSuggestion`) fires Рђћ a РюЁ/РЮї suggestion is posted in the group for calendar add
- [ ] **MM-03**: On successful high-confidence extraction, the bot posts a single-line discreet ack: `"­ЪЊї noted: {type} Рђћ {summary}"` in the group's language. Low-confidence extractions are silently discarded. Vision API errors are logged but produce no group message

### Phase 53 Рђћ Smarter Search (Restaurants)

- [ ] **SRCH2-01**: `travelParser` has a `queryType='restaurants'` branch triggered by Hebrew and English restaurant keywords. `travelSearch.geminiMapsSearch` requests restaurant-specific fields (photo_url, open_now, price_level, cuisine, reservation_url). `travelFormatter` produces compact one-liner results: `­ЪЇй№ИЈ {name} ┬и {cuisine} ┬и {price_tier} ┬и {open_now} ┬и {rating}РГљ ┬и {url}`. Transit, attractions, and shopping search are explicitly out of scope for v2.1

### Phase 54 Рђћ Proactive Day-Of Intelligence

- [ ] **DAY-01**: A 15-min cron posts at most one briefing per active-trip per day, at `briefing_time` destination-tz (default 08:00), within the window `[start_date Рѕњ 1day, end_date]`. Archived trips are skipped. `metadata.last_briefing_date` prevents duplicate posts
- [ ] **DAY-02**: Briefings enrich the calendar event list with: weather from OpenWeather (destination coords cached in `metadata.coords`); transit alerts from Gemini grounded search (`googleSearch` tool); unresolved open questions from DB; today's conflicts from `trip_decisions.conflicts_with`; budget burn from `getBudgetRollup`. Final composition by Gemini with structured input + Hebrew output instruction
- [ ] **DAY-03**: On any enrichment source failure (OpenWeather timeout, Gemini error, etc.), the briefing falls back to a minimal calendar-only template posted in Hebrew. No day is skipped because of enrichment failure

### Phase 55 Рђћ Trip Dashboard View

- [ ] **DASH-TRIP-01**: Dashboard route `/trips/:groupJid` renders trip header (destination, dates, countdown, budget target vs actual per category), vertical timeline of confirmed calendar events, Leaflet map with markers per decision that has lat/lng, decisions board grouped by category with filter-by-origin control, open-questions list, and conflict alerts. Archived trips render in read-only mode
- [ ] **DASH-TRIP-02**: Minimal-edit controls wired through JWT-gated API routes: `DELETE /api/trips/:groupJid/decisions/:id` (soft-delete via `status='deleted'`), `PATCH /api/trips/:groupJid/questions/:id/resolve`, `PATCH /api/trips/:groupJid/budget`. SSE broadcasts propagate changes to every open dashboard session within ~3s
- [ ] **DASH-TRIP-03**: Dashboard "Export" button calls `POST /api/trips/:groupJid/export` which creates a Google Doc titled `{destination} {start_date} Рђћ Trip Summary` containing trip header, timeline, decisions grouped by category, open questions, and budget summary. Uses existing Google OAuth with a new `https://www.googleapis.com/auth/documents` scope (one-time re-auth by owner during Phase 55 UAT). Returns owner-private shareable URL
