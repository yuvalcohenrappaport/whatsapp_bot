<task type="auto">
  <name>Task 2: Add invalid date validation to dateExtractor</name>
  <files>src/groups/dateExtractor.ts</files>
  <action>
In `dateExtractor.ts`, add a validation step after `new Date(d.date)` to filter out invalid dates. This is a minimal one-line patch because Phase 19 will modify dateExtractor.ts for the suggest-then-confirm flow.

**Current behavior (lines 100-104):**
The `highConfidence` array is mapped to `ExtractedDate` objects using `new Date(d.date)`. If Gemini returns a malformed date string (e.g., "next Tuesday" instead of ISO 8601), `new Date()` produces an `Invalid Date` object that would propagate to `createCalendarEvent()`.

**New behavior:**
After mapping to `ExtractedDate` objects, filter out any entries where `date.getTime()` returns `NaN` (which indicates an Invalid Date). Log a warning for any filtered-out entries.

**Specific change:**

Replace the final return statement (the `return highConfidence.map(...)` block) with:

1. Map `highConfidence` to `ExtractedDate[]` as before (using `new Date(d.date)`)
2. Filter the mapped results: keep only entries where `!isNaN(date.getTime())`
3. For any filtered-out entries, log a warning with the original date string from Gemini: `logger.warn({ dateStr: d.date, title: d.title }, 'Invalid date from Gemini ŌĆö skipping')`
4. Return the filtered array

This can be done cleanly by changing the return block to:

```
const mapped = highConfidence.map((d) => ({
  title: d.title,
  date: new Date(d.date),
  confidence: d.confidence,
  _raw: d.date,  // keep for logging
}));

const valid = mapped.filter((d) => {
  if (isNaN(d.date.getTime())) {
    logger.warn({ dateStr: d._raw, title: d.title }, 'Invalid date from Gemini ŌĆö skipping');
    return false;
  }
  return true;
});

return valid.map(({ title, date, confidence }) => ({ title, date, confidence }));
```

**Do NOT change:**
- The Zod schema or system prompt
- The `hasNumberPreFilter` function
- The function signature or return type
- Any other file

  </verify>

The `extractDates` function filters out entries where Gemini returned a malformed date string. Invalid dates are logged as warnings and excluded from calendar event creation.
