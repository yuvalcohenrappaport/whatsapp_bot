```typescript
export interface PreviewItem {
  task: string;
  contactName: string | null;
  snippet: string; // pre-truncated, max 100 chars
}

export function composePreview(
  items: PreviewItem[],
  language: 'he' | 'en',
  contactName: string | null, // source-chat name for batch header
): string {
  if (items.length === 0) throw new Error('empty preview items');
  if (items.length === 1) return composeSingle(items[0], language);
  return composeBatched(items, language, contactName);
}
```

**Single-item EN:**
```
📝 <task>
👤 <name>
💬 "<snippet>"
Reply ✅ / ❌ / edit: <text>
```

**Single-item HE:** mirror with Hebrew glyphs:
```
📝 <task>
👤 <name>
💬 "<snippet>"
השב ✅ / ❌ / עריכה: <טקסט>
```

**Batched EN (N > 1):**
```
📝 <N> items from <name>:
1. <task1>
   💬 "<snippet1>"
2. <task2>
   💬 "<snippet2>"
...

Reply: `1 ✅` / `2 ❌` / `3 edit: <text>` (or `✅` to approve all)
```

**Batched HE:** mirror. Use `פריטים` for "items", `השב` for "Reply", `לאישור הכל` for "to approve all".

Single-item omits item numbering and snippet/name-header deduplication. Snippets pre-truncated by caller (keep composer dumb).

If `contactName` is null in batched mode, header says `N items:` (no "from X") in both languages.

Include a single trailing newline. No Markdown.

**Synonym tables** (module-level const):

```typescript
const APPROVE_SYNONYMS = new Set([
  '✅', '✓', 'approve', 'ok', 'yes', 'y', 'אישור', 'כן',
]);
const REJECT_SYNONYMS = new Set([
  '❌', '✗', 'reject', 'no', 'n', 'ביטול', 'לא',
]);
const EDIT_PREFIXES = ['edit:', 'עריכה:']; // case-insensitive match for EN; HE literal
```

**Parsing algorithm:**

1. Trim. If empty → return `[]`.
2. Check un-numbered bulk approve: if trimmed text (case-insensitive for Latin) matches an APPROVE_SYNONYMS entry → return `[{action: 'approve', itemIndex: 'all'}]`. Same for REJECT.
3. Otherwise split the reply into directives. A directive starts with a leading integer `N` (itemIndex, 1-based, must be ≥ 1 and ≤ itemCount), followed by whitespace, followed by:
   - an approve synonym → directive
   - a reject synonym → directive
   - an edit prefix + the rest of the text until the next directive boundary or end-of-string → directive
4. Multiple directives in one reply: greedy match, each starting with `N ` where `N` is a 1-digit integer (1–9 sufficient for the 2-min debounce bucket; if you hit 10+ items in a bucket, behavior is documented as single-digit-only).
5. If ANY directive fails to parse (invalid synonym, out-of-range index, malformed edit) → return `[]`. Caller is responsible for re-posting the grammar hint. No partial parse.
6. Multiple directives with the same item index are permitted in the returned array (caller decides precedence; downstream dedupes to last-wins).

**Examples the tests must cover:**

| Input (itemCount=3) | Expected parse |
|---|---|
| `✅` | `[{approve, all}]` |
| `❌` | `[{reject, all}]` |
| `1 ✅` | `[{approve, 1}]` |
| `1 ✅ 2 ❌ 3 edit: Send report to Lee` | 3 directives |
| `1 yes` | `[{approve, 1}]` |
| `1 לא` | `[{reject, 1}]` |
| `1 עריכה: שלח את הדוח` | `[{edit, 1, editText: 'שלח את הדוח'}]` |
| `1 ✅ banana` | `[]` (trailing garbage unparseable) |
| `5 ✅` (itemCount=3) | `[]` (out of range) |
| `edit: no number` | `[]` (no item index and no un-numbered edit grammar) |
| `` (empty) | `[]` |

**Notes:**
- Un-numbered `edit:` is NOT supported — edit always requires an item number, since it needs to know which actionable to rewrite.
- Case insensitivity applies only to Latin tokens (approve/reject synonyms, `edit:` prefix). Hebrew and emoji tokens match literally.
- `editText` is trimmed but preserves internal whitespace and colons.
