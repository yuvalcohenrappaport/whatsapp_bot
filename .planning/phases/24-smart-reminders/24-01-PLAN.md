<task type="auto">
  <name>Task 2: Two-tier scheduler, service orchestration, and handleOwnerCommand wiring</name>
  <files>src/reminders/reminderScheduler.ts, src/reminders/reminderService.ts, src/pipeline/messageHandler.ts, src/index.ts</files>
  <action>
1. Create `src/reminders/reminderScheduler.ts`:
   - Private `activeTimers = new Map<string, NodeJS.Timeout>()` for in-memory timer tracking
   - Export `scheduleReminder(id: string, fireAt: number, onFire: (id: string) => void): void`:
     - Calculate delay = fireAt - Date.now()
     - If delay <= 0: call onFire immediately, return
     - If delay > 24 * 60 * 60 * 1000 (24h): skip (leave for hourly scan), return
     - Otherwise: set setTimeout, store in activeTimers map
     - On fire: delete from map, call onFire callback
   - Export `cancelScheduledReminder(id: string): void` — clearTimeout from map, delete entry
   - Export `startHourlyScan(onFire: (id: string) => void): void`:
     - setInterval at 3600_000 ms
     - Query getRemindersInWindow(Date.now(), Date.now() + 24h)
     - For each not already in activeTimers: call scheduleReminder
   - Export `getActiveTimerCount(): number` — returns activeTimers.size (for debugging/dashboard)
   - Export `scheduleAllUpcoming(onFire: (id: string) => void): void` — queries reminders in 24h window and schedules them (used on startup)

2. Create `src/reminders/reminderService.ts`:
   - Import parser, scheduler, DB queries, sock from api/state, config
   - Internal `fireReminder(id: string)` function:
     - Get reminder from DB by id
     - If status !== 'pending', return (belt-and-suspenders check)
     - Update status to 'fired' in DB
     - Send WhatsApp message: `sock.sendMessage(config.USER_JID, { text: '🔔 Reminder: {task}' })`
   - Export `tryHandleReminder(sock: WASocket, text: string): Promise<boolean>`:
     - If !hasReminderIntent(text): return false
     - Call parseReminderCommand(text)
     - If result is null or intent is 'none': return false
     - If intent is 'set':
       - Parse dateTime string to Unix ms (use new Date(dateTime).getTime())
       - If no dateTime: compute tomorrow 9am IST
       - Generate UUID, call insertReminder
       - Determine routing: hoursUntil = (fireAt - Date.now()) / 3600_000
       - If hoursUntil <= 24: schedule setTimeout via scheduleReminder(id, fireAt, fireReminder)
       - If hoursUntil > 72: create Google Calendar event via createPersonalCalendarEvent AND store for WhatsApp delivery
       - If 24 < hoursUntil <= 72: create Google Calendar event only
       - Store calendarEventId in DB if calendar event created
       - Format confirmation: "⏰ {task} — {formatted time}" using Intl.DateTimeFormat for Asia/Jerusalem locale
       - Send confirmation to self-chat, return true
     - If intent is 'cancel' or 'edit': return true (stub — Plan 02 implements full cancel/edit logic). For now, send "Cancel/edit support coming soon." to self-chat.
   - Export `initReminderSystem(sock: WASocket): void`:
     - Start hourly scan: startHourlyScan(fireReminder)
     - Schedule all upcoming reminders: scheduleAllUpcoming(fireReminder)
     - Log initialization

3. Wire into `src/pipeline/messageHandler.ts`:
   - Import `tryHandleReminder` from '../reminders/reminderService.js'
   - In `handleOwnerCommand`, AFTER the calendar approval stanzaId check block (line ~143), BEFORE the snooze command parsing:
     ```
     const reminderHandled = await tryHandleReminder(sock, text);
     if (reminderHandled) return true;
     ```

4. Wire into `src/index.ts`:
   - Import `initReminderSystem` from './reminders/reminderService.js'
   - Call `initReminderSystem(getState().sock!)` inside the `onOpen` callback (after updateState), so it runs after socket connects. This ensures sock is available for firing reminders.
   - Alternatively, call it right after `initReminderScheduler()` in main() if sock isn't needed at init time (scheduler just sets timers, firing uses getState().sock).

   The better approach: call `initReminderSystem()` (no sock arg) in main() after initReminderScheduler(). Have fireReminder get sock from getState().sock at fire time (lazy, like other message senders). This way the scheduler starts immediately and timers are set, but firing waits for connection.
