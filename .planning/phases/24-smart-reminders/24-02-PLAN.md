   - Log count of recovered/skipped reminders

4. Ensure `initReminderSystem` is called in index.ts `onOpen` callback (or in main after socket connects). Recovery needs sock to send messages, so it must run after connection is established. Update the init call location if Plan 01 placed it too early.

   - On intent 'cancel':
     - Get all pending reminders from DB
     - If none: send "No pending reminders to cancel." Return true.
     - Call matchReminderForCancelEdit with editTarget and pending list
     - If exactly 1 match:
       - Call cancelScheduledReminder(id) to clear setTimeout
       - Call updateReminderStatus(id, 'cancelled')
       - If reminder has calendarEventId, log that calendar event remains (don't delete calendar events — too complex for v1, user can delete manually)
       - Send confirmation: "❌ Cancelled: {task}"
     - If multiple matches:
       - Send numbered list: "Multiple reminders match. Reply with the number:\n{1. task — time\n2. task — time\n...}"
       - Store the pending disambiguation state (use a module-level variable `pendingCancelIds: Map<number, string>` that maps choice numbers to reminder IDs)
       - On next message in self-chat, if it's a single digit and pendingCancelIds has entries, treat as disambiguation reply
     - If no matches: send "No matching reminder found. Your pending reminders:\n{list}"
     - Return true

3. Implement edit handling in reminderService.ts:
   - On intent 'edit':
     - Same matching flow as cancel
     - If exactly 1 match:

       - Send confirmation: "✏️ Updated: {task} — {new time}"
     - If multiple matches: same disambiguation as cancel
     - If no matches: same as cancel
     - Return true
