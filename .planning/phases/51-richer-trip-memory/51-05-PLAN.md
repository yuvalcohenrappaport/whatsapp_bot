<objective>
Register a daily 02:00 Asia/Jerusalem cron that archives expired trips (`end_date + 3 days < now`). For each expired row: move trip_contexts row to trip_archive (fresh UUID + archived_at), then flip archived=1 on every trip_decision for that group.

Purpose: MEM2-05 — without auto-archive, active-trip queries would forever include finished trips, eventually polluting classifier context + dashboard + day-of briefings. No manual un-archive path (deferred — see CONTEXT §Out-of-Scope).

Output: new cron module, one-line wire-up in `src/index.ts`, tests.
</objective>

<decision_rationale>
**Decision-archival ordering: move context FIRST, then flip decisions.**

If we flipped decisions before moving the context and the process crashed in between, we'd have a trip_contexts row pointing at decisions that all report themselves archived — reads would see an "active trip with no decisions." Moving context first means a mid-run crash leaves the trip already archived (dashboard sees it in archive) with decisions still joinable via group_jid to the trip_archive row by name. Safer failure mode.

**No transaction wrapping the whole operation:** better-sqlite3 transactions across multiple query calls are supported, but the archive cron isn't latency-sensitive and we'd rather keep per-group operations atomic independently. Each group is its own unit — a crash mid-list leaves some groups archived and some not; next run picks up the remainder. See test case "partial crash resumes cleanly."

**Registered in `src/scheduler/` (not `src/cron/`):** project convention — existing `scheduledMessageScheduler.ts` lives in `src/scheduler/`, and `src/cron/` does not exist. Keep convention.
</decision_rationale>

    ```ts
    import * as cron from 'node-cron';
    import pino from 'pino';
    import { config } from '../config.js';
    import {
      getExpiredActiveContexts,
      moveContextToArchive,
      markDecisionsArchivedForGroup,
    } from '../db/queries/tripMemory.js';

    const logger = pino({ level: config.LOG_LEVEL });

    let scheduled: cron.ScheduledTask | null = null;

    /**
     * Archive all expired trip contexts (end_date + 3 days < now).
     * Per-group atomic: context moved first, then decisions flipped.
     * Mid-run crash leaves remaining groups for next run to pick up.
     * Returns count of archived groups for logging + tests.
     */
    export function runArchiveTripsOnce(nowMs = Date.now()): { archivedCount: number } {
      const expired = getExpiredActiveContexts(nowMs);
      let archivedCount = 0;

      for (const row of expired) {
        try {
          const result = moveContextToArchive(row.groupJid);
          if (!result) {
            logger.warn({ groupJid: row.groupJid }, 'moveContextToArchive returned null — skipping');
            continue;
          }
          const flipped = markDecisionsArchivedForGroup(row.groupJid);
          archivedCount++;
          logger.info(
            { groupJid: row.groupJid, archiveId: result.archiveId, decisionsFlipped: flipped },
            'Trip archived',
          );
        } catch (err) {
          logger.error({ err, groupJid: row.groupJid }, 'Failed to archive trip — will retry next run');
          // Continue — other expired trips should still process
        }
      }

      if (archivedCount > 0) {
        logger.info({ archivedCount }, 'Archive trips cron run complete');
      }
      return { archivedCount };
    }

    /**
     * Register the daily 02:00 Asia/Jerusalem cron.
     * Idempotent — calling twice replaces the existing job (supports dev hot-reload).
     */
    export function initArchiveTripsCron(): void {
      if (scheduled) scheduled.stop();

      scheduled = cron.schedule(
        '0 2 * * *', // daily at 02:00
        () => {
          try {
            runArchiveTripsOnce();
          } catch (err) {
            logger.error({ err }, 'Archive trips cron handler threw');
          }
        },
        { timezone: 'Asia/Jerusalem' },
      );
      logger.info('Archive trips cron initialized (daily 02:00 Asia/Jerusalem)');
    }
    ```
