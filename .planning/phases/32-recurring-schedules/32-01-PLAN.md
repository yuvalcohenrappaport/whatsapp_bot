    1. Create `src/scheduler/cronUtils.ts` with two exported functions:

    **buildCronExpression(cadence: 'daily' | 'weekly' | 'monthly', scheduledAtMs: number, timezone?: string): string**
    - Default timezone: 'Asia/Jerusalem'
    - Use `Intl.DateTimeFormat` with the timezone to extract local hour, minute, day-of-month, and weekday from scheduledAtMs
    - daily: `${minute} ${hour} * * *`
    - weekly: `${minute} ${hour} * * ${weekday}` (0=Sun..6=Sat)
    - monthly: `${minute} ${hour} ${day} * *`
    - See Pattern 1 in RESEARCH.md for exact implementation

    **getNextOccurrence(cronExpr: string, tz?: string): number | null**
    - Default tz: 'Asia/Jerusalem'
    - Parse 5-field cron: parts[0]=minute, parts[1]=hour, parts[2]=day-of-month (* or number), parts[4]=day-of-week (* or 0-6)
    - Loop daysAhead from 0 to 400:
      - Use `Intl.DateTimeFormat` to get local year/month/day/weekday for each probe date
      - Skip if day-of-month doesn't match (when not *)
      - Skip if day-of-week doesn't match (when not *)
      - For matching day, find UTC ms for that date at hour:minute in tz by probing offsets -840..+840
      - Return first utcMs that is > now + 60_000 (1 minute buffer)
    - Return null if no match in 400 days
    - CRITICAL: Do NOT use node-cron getNextRun() — it has a documented bug for weekday expressions in v4.2.1 (returns dates years in the future). See Pitfall 1 in RESEARCH.md.

    Also export a helper **getCadenceFromCron(cronExpression: string | null): 'daily' | 'weekly' | 'monthly' | null**:
    - null input -> null
    - parts[2] !== '*' -> 'monthly'
    - parts[4] !== '*' -> 'weekly'
    - else -> 'daily'

      if (fresh?.cronExpression && fresh.status !== 'cancelled') {

        if (nextMs) {
