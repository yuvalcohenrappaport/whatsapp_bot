  <action>
    Part A -- FTS5 migration file:

    Create `drizzle/0010_fts5_group_messages.sql` as a hand-written migration (not generated by Drizzle). Contents must be valid SQL with Drizzle-style `statement-breakpoint` markers between statements:

    1. CREATE VIRTUAL TABLE IF NOT EXISTS group_messages_fts USING fts5(body, content='group_messages', content_rowid='rowid')
    2. Three triggers for sync:
       - `group_messages_fts_ai` AFTER INSERT: inserts new.rowid and new.body into FTS
       - `group_messages_fts_ad` AFTER DELETE: deletes old.rowid from FTS using the FTS5 delete command (INSERT INTO group_messages_fts(group_messages_fts, rowid, body) VALUES('delete', old.rowid, old.body))
       - `group_messages_fts_au` AFTER UPDATE: delete old + insert new
    3. Rebuild command: INSERT INTO group_messages_fts(group_messages_fts) VALUES('rebuild') -- indexes all existing rows

    Each statement separated by `--> statement-breakpoint` on its own line. Use `CREATE TRIGGER IF NOT EXISTS` for all triggers.

    Add an entry to `drizzle/meta/_journal.json` for index 10 with tag `0010_fts5_group_messages`, version "6", breakpoints true. Generate a `when` timestamp using Date.now().

    Part B -- tripMemory.ts query module:

    Create `src/db/queries/tripMemory.ts` following the patterns in `calendarEvents.ts` and `groupMessages.ts`. Import `db` from `'../client.js'`, import `sql` from `'drizzle-orm'`, import `eq` from `'drizzle-orm'`, import `tripContexts` and `tripDecisions` from `'../schema.js'`.

    Export these functions:

    1. `getTripContext(groupJid: string)` -- select all columns from tripContexts where groupJid matches, return `.get()` (single row or undefined)

    2. `upsertTripContext(groupJid: string, data: { destination?: string | null; dates?: string | null; contextSummary?: string | null })` -- use `db.insert(tripContexts).values({ groupJid, ...data, lastClassifiedAt: Date.now(), updatedAt: Date.now() }).onConflictDoUpdate({ target: tripContexts.groupJid, set: { ...data, lastClassifiedAt: Date.now(), updatedAt: Date.now() } }).run()`

    3. `getDecisionsByGroup(groupJid: string, type?: string)` -- select all from tripDecisions where groupJid matches, optionally also filter by type if provided, order by createdAt desc, return `.all()`

    4. `insertTripDecision(decision: { id: string; groupJid: string; type: string; value: string; confidence: string; sourceMessageId: string | null })` -- insert into tripDecisions, return `.run()`

    5. `getUnresolvedOpenItems(groupJid: string)` -- select from tripDecisions where groupJid matches AND type equals 'open_question' AND resolved equals false, order by createdAt desc, return `.all()`

    6. `resolveOpenItem(decisionId: string)` -- update tripDecisions set resolved=true where id matches, return `.run()`

    7. `searchGroupMessages(groupJid: string, query: string, limit = 10)` -- FTS5 search using raw SQL via `db.all(sql\`...\`)`. Sanitize the query: split on whitespace, filter words shorter than 2 chars, wrap each remaining word in double quotes (strip any existing quotes first), join with spaces. If sanitized result is empty, return empty array. Execute an FTS5 MATCH query joining group_messages_fts with group_messages on rowid, filtering by group_jid, ordering by fts.rank, limited to `limit`. Return array of objects with id, senderName, body, timestamp fields (camelCase -- map from snake_case column names).

    Use the `sql` tagged template from drizzle-orm for the FTS5 query (same as the pattern in the research doc). The return type annotation should be explicit.
