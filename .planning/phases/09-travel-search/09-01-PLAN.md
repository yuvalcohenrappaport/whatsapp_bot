**1. Create `src/groups/travelParser.ts` — Gemini structured intent parsing:**

Follow the exact pattern from `dateExtractor.ts` (same project — `zod/v3` import, `zodToJsonSchema`, Gemini `responseMimeType: 'application/json'`).

Define and export `TravelIntent` type and `TravelIntentSchema`:

```typescript
import { z } from 'zod/v3';
import zodToJsonSchema from 'zod-to-json-schema';
```

Schema fields:
- `isTravelRelated: z.boolean()` — true if this is a travel search request
- `isVague: z.boolean()` — true if essential info (at minimum destination) is missing
- `clarificationQuestion: z.string().nullable()` — question to ask if vague, in the detected language; null if clear
- `queryType: z.enum(['flights', 'hotels', 'restaurants', 'activities', 'car_rental', 'general']).nullable()`
- `searchQuery: z.string().nullable()` — optimized Google search query string (Gemini builds this)
- `destination: z.string().nullable()`
- `dates: z.string().nullable()`
- `budget: z.string().nullable()`
- `preferences: z.string().nullable()`

Export `type TravelIntent = z.infer<typeof TravelIntentSchema>;`

Create `parseTravelIntent(messageText: string, recentGroupContext: string, lang: 'he' | 'en'): Promise<TravelIntent | null>`:
- System instruction tells Gemini: "You are a travel assistant parsing @mention requests from a WhatsApp group. Determine if the message is travel-related. If yes, extract all available details and build an optimized Google search query. If essential info is missing (especially destination), set isVague=true and provide a clarification question in {lang}. Consider the recent group context to infer missing details. Respond in JSON."
- Include `recentGroupContext` in the user content alongside the message text
- Use same Gemini call pattern as dateExtractor.ts
- Return parsed TravelIntent or null on error (never crash the pipeline)

**2. Create `src/groups/travelHandler.ts` — @mention detection and pipeline orchestration:**

Import `getState` from state.ts, `parseTravelIntent` from travelParser.ts, `detectGroupLanguage` from groupMessagePipeline.ts (export it — see step 3), `getGroupMessagesSince` from db queries.

**Bot mention detection function** `isBotMentioned(body, mentionedJids, botJid, botDisplayName)`:
- Check native mention: `mentionedJids.some(jid => jid.split('@')[0] === botJid.split('@')[0])` — match on numeric prefix only (handles LID format mismatch per research pitfall 2)
- Check text mention: if `botDisplayName` is set, check `body.toLowerCase().includes(botDisplayName.toLowerCase())` — matches both `@BotName` and plain `BotName` typed in text
- Return true if either matches

**Main handler** `handleTravelMention(groupJid, msg, quotedMessageId, mentionedJids)`:
- Get `botJid` and `botDisplayName` from `getState()`
- If either is null, log warning and return false (socket not connected yet)
- Call `isBotMentioned()` — if false, return false (not a mention, caller continues to debounce)
- If mention detected:
  - Send "Searching..." / "מחפש..." indicator immediately via `sock.sendMessage(groupJid, { text })` — detect language first using `detectGroupLanguage(groupJid)`. Per locked decision: this fires BEFORE Gemini parsing starts.
  - Gather recent group context: `getGroupMessagesSince(groupJid, Date.now() - 2 * 60 * 60 * 1000, 20)` — last 2 hours, up to 20 messages. Join bodies with newlines as context string.
  - Call `parseTravelIntent(msg.body, recentContext, lang)`
  - If parsing returns null or `isTravelRelated === false`:
    - Send casual help text in detected language. English example: "Hey! I can help you find travel deals. Try mentioning me with something like:\n\n@BotName flights to Rome next week\n@BotName hotels in Barcelona March 10-15\n@BotName restaurants near the Eiffel Tower\n\nI'll search and share the best options right here!" (use actual botDisplayName in examples). Hebrew equivalent for 'he'.
    - Return true (mention was handled)
  - If `isVague === true` and `clarificationQuestion` is not null:
    - Send the clarification question text in the group
    - Return true
  - If `isTravelRelated === true` and `isVague === false`:
    - For now (Plan 01), log the parsed intent and send a placeholder message: "Travel search coming soon! Query: {searchQuery}" (Plan 02 will replace this with actual search + format)
    - Export the TravelIntent so Plan 02 can use it
    - Return true
  - Return true (all @mention paths handled)
