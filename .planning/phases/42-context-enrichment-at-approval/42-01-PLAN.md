Create `src/approval/enrichmentService.ts`. Follow the `CommitmentDetectionService.extractCommitments` pattern (Zod schema → generateJson → safeParse → try/catch → fallback).

Public API:
```typescript
export interface Enrichment { title: string; note: string }
export async function enrichActionable(actionable: Actionable): Promise<Enrichment>
```

Behavior:

1. **user_command short-circuit** — at the top of `enrichActionable`:
   ```typescript
   if (actionable.sourceType === 'user_command') {
     return { title: actionable.task, note: buildBasicNote(actionable) };
   }
   ```
   Imports `buildBasicNote` from `./approvalHandler.js` (Task 2 exports it). NO Gemini call. Rationale: Phase 41 Q9A-1 decision documented in 41-04-SUMMARY — `/remind me` commands are already owner-authored, enrichment adds latency for zero gain.

2. **Load chat history** — `const recent = await getRecentMessages(actionable.sourceContactJid, 10);` from `src/db/queries/messages.js`. Already returns chronological order, {body, fromMe, timestamp, ...}. If `recent.length === 0`, still call Gemini with an empty history — the trigger message + contact name may be enough. (Do NOT short-circuit to fallback just because history is empty.)

3. **Zod schema** (mirror the CommitmentDetectionService style):
   ```typescript
   const EnrichmentSchema = z.object({
     title: z.string().min(1).max(200).describe(
       'Self-contained Google Tasks title. MUST include the contact\'s name when not self-chat. Resolve pronouns/vague refs using the chat context. Include a concrete deadline (e.g. "by Monday") when a time was detected. Max ~80 chars preferred.'
     ),
     note: z.string().min(1).describe(
       'Rich audit note. Multi-line. Include: Contact: <name>, Original: "<trigger message>", Context: <1-2 line summary of the chat context that makes the task self-explanatory>.'
     ),
   });
   const ENRICHMENT_JSON_SCHEMA = z.toJSONSchema(EnrichmentSchema);
   ```

4. **System prompt** (English — Gemini follows it regardless of chat language; the produced title.language should match the chat's detectedLanguage):
   ```
   You enrich detected commitments/tasks into self-contained Google Tasks entries.

   You receive:
   - A detected task string (often vague, e.g. "Check it")
   - The contact's display name and JID
   - The language the chat is in (he or en)
   - The most recent ~10 messages from that chat, chronological

   Your output is a JSON object {title, note}:

   title rules:
   - Self-contained — readable with zero chat context
   - Include the contact's name when sourceContactJid is NOT self-chat (the `self@s.whatsapp.net` / `<USER_JID>` case)
   - Resolve pronouns and vague refs ("it", "that", "the thing") using the chat history
   - Include a concrete deadline when one was detected ("by Monday", "tomorrow evening", "next week")
   - Write the title in the same language as the chat (he or en)

   note rules:
   - Multi-line, for audit from Google Tasks UI alone
   - Line 1: `Contact: <name>`
   - Line 2: `Original: "<verbatim trigger message, truncated to 200 chars>"`
   - Line 3+: `Context: <1-2 line summary of the chat context that makes the task self-explanatory>`

   If you cannot confidently enrich, return a title that falls back to the detected task VERBATIM plus the contact name, and a basic note. Never invent facts not in the provided messages.
   ```

5. **User content** — a single formatted block:
   ```
   Detected task: <actionable.task>
   Contact name: <actionable.sourceContactName ?? '(unknown)'>
   Contact JID: <actionable.sourceContactJid>
   Chat language: <actionable.detectedLanguage>
   Trigger message: "<actionable.sourceMessageText>"

   Recent chat history (chronological, `them:` = contact, `me:` = user):
   <for each m in recent: `${m.fromMe ? 'me' : 'them'}: ${m.body}`>
   ```
   If `recent.length === 0`, replace the history block with `(no prior messages available)`.

6. **Call + validate**:
   ```typescript
   try {
     const raw = await generateJson<{title: string; note: string}>({
       systemPrompt,
       userContent,
       jsonSchema: ENRICHMENT_JSON_SCHEMA as Record<string, unknown>,
       schemaName: 'actionable_enrichment',
     });
     if (!raw) return fallback(actionable); // Gemini returned null
     const parsed = EnrichmentSchema.safeParse(raw);
     if (!parsed.success) {
       logger.warn({err: parsed.error.message, id: actionable.id}, 'enrichment Zod parse failed — using fallback');
       return fallback(actionable);
     }
     if (!parsed.data.title.trim()) return fallback(actionable);
     return { title: parsed.data.title.trim(), note: parsed.data.note.trim() };
   } catch (err) {
     logger.warn({err, id: actionable.id}, 'enrichment Gemini call threw — using fallback');
     return fallback(actionable);
   }
   ```

7. **Fallback** — module-local:
   ```typescript
   function fallback(a: Actionable): Enrichment {
     return { title: a.task, note: buildBasicNote(a) };
   }
   ```

   ```typescript
   async function approveAndSync(sock, actionable) {
     // 1. Flip status FIRST — enrichment failure must NEVER block approval (APPR-02 behavior preserved).
     updateActionableStatus(actionable.id, 'approved');

     // 2. Enrich (Gemini + fallback — never throws).
     const enrichment = await enrichActionable(actionable);

     // 3. Persist enrichment on the actionable row.
     updateActionableEnrichment(actionable.id, {title: enrichment.title, note: enrichment.note});

     // 4. Push to Google Tasks with the ENRICHED payload (title + note).
     if (isTasksConnected()) {
       try {
         const result = await createTodoTask({title: enrichment.title, note: enrichment.note});
         updateActionableTodoIds(actionable.id, {todoTaskId: result.taskId, todoListId: result.listId});
       } catch (err) {
         logger.warn({err, id: actionable.id}, 'Google Tasks push failed on approval — actionable stays approved+enriched in DB');
       }
     } else {
       logger.info({id: actionable.id}, 'Google Tasks not connected — approved actionable not synced');
     }

     // 5. Confirm to owner (shape unchanged from Phase 41 — still shows actionable.task, not enriched title;
     //    the enriched title is what the user will see in Google Tasks, the self-chat confirmation stays
     //    compact and matches what they saw in the preview).
     await sock.sendMessage(config.USER_JID, {text: approvedConfirmation(actionable)});
   }
   ```
