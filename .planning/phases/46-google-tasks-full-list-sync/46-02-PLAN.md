```ts

  items: CalendarItem[];

    tasks: SourceStatus;
    events: SourceStatus;
    linkedin: SourceStatus;
    gtasks: SourceStatus;    // NEW
  };
};
```

### 4. Extend fetchCalendarWindow

Add gtasks as the fourth slot in `Promise.allSettled`:
```ts
const [tasksRes, eventsRes, linkedinRes, gtasksRes] = await Promise.allSettled([

  fetchGtasksCalendarItems(fromMs, toMs),

```

In the result assembly, add:
```ts
const sources = { tasks: 'ok', events: 'ok', linkedin: 'ok', gtasks: 'ok' } as CalendarEnvelope['sources'];

// existing tasks/events/linkedin handling unchanged...

  items.push(...gtasksRes.value);
} else {
  sources.gtasks = 'error';
}
```

### 5. Extend hashCalendarEnvelope

Change status bits to include gtasks:
```ts
const statusBits = `${env.sources.tasks}:${env.sources.events}:${env.sources.linkedin}:${env.sources.gtasks}`;
```

### 6. No new routes registered here

The per-source `/api/google-tasks/items` route is already in googleTasks.ts (Plan 46-01). The unified `/api/calendar/items` and `/api/calendar/stream` routes are unchanged structurally.
