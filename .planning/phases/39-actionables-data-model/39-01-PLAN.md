  <name>Task 1: Add `actionables` table to src/db/schema.ts</name>
  <files>src/db/schema.ts</files>
  <action>
Append the following table definition after the existing `todoTasks` block (around line 252):

```typescript
export const actionables = sqliteTable(
  'actionables',
  {
    id: text('id').primaryKey(), // UUID
    sourceType: text('source_type').notNull(), // 'commitment' | 'task' | 'user_command'
    sourceContactJid: text('source_contact_jid').notNull(), // for user_command this is config.USER_JID
    sourceContactName: text('source_contact_name'), // snapshot at detection time, nullable
    sourceMessageId: text('source_message_id'), // WhatsApp message id of trigger (null for user_command)
    sourceMessageText: text('source_message_text').notNull().default(''), // full trigger text snapshot
    detectedLanguage: text('detected_language').notNull().default('en'), // 'he' | 'en'
    originalDetectedTask: text('original_detected_task').notNull(), // IMMUTABLE — what Gemini/user initially wrote
    task: text('task').notNull(), // mutable, replaced by `edit: <text>` reply grammar
    status: text('status').notNull().default('pending_approval'), // 'pending_approval' | 'approved' | 'rejected' | 'fired' | 'expired'
    detectedAt: integer('detected_at').notNull(),
    fireAt: integer('fire_at'), // Unix ms, nullable for timeless tasks
    enrichedTitle: text('enriched_title'), // populated by Phase 42 on approval
    enrichedNote: text('enriched_note'), // populated by Phase 42 on approval
    todoTaskId: text('todo_task_id'), // Google Tasks id, set on approval+sync
    todoListId: text('todo_list_id'), // Google Tasks list id
    approvalPreviewMessageId: text('approval_preview_message_id'), // WhatsApp self-chat msg id for quoted-reply matching (Phase 41)
    createdAt: integer('created_at')
      .notNull()
      .$defaultFn(() => Date.now()),
    updatedAt: integer('updated_at')
      .notNull()
      .$defaultFn(() => Date.now()),
  },
  (table) => [
    index('idx_actionables_status_detected').on(table.status, table.detectedAt),
    index('idx_actionables_preview_msg').on(table.approvalPreviewMessageId),
  ],
);
```

The column-name ↔ TS-name mapping follows the existing `scheduled_messages` convention (snake_case in SQL, camelCase in TS). Do NOT modify the existing `reminders` or `todoTasks` tables in this plan.
