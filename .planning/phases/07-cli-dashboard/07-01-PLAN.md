  <action>
1. Install new dependencies:

2. Create `cli/db.ts` — a minimal Drizzle client that does NOT import `src/config.ts` (which would fail on missing GEMINI_API_KEY etc.). Instead, read DB_PATH directly from `process.env.DB_PATH` with a default of `./data/bot.db`. Resolve the path relative to the project root (use `import.meta.url` to derive `__dirname`, then resolve). Set pragmas: `journal_mode = WAL` and `busy_timeout = 5000`. Import schema from `../src/db/schema.js`. Export `db` (drizzle instance). Do NOT call `migrate()`.

   Key code structure:
   ```typescript
   import Database from 'better-sqlite3';
   import { drizzle } from 'drizzle-orm/better-sqlite3';
   import * as schema from '../src/db/schema.js';
   import { fileURLToPath } from 'node:url';
   import path from 'node:path';

   const __dirname = path.dirname(fileURLToPath(import.meta.url));
   const PROJECT_ROOT = path.resolve(__dirname, '..');
   const DB_PATH = process.env.DB_PATH
     ? path.resolve(process.env.DB_PATH)
     : path.join(PROJECT_ROOT, 'data', 'bot.db');

   const sqlite = new Database(DB_PATH);
   sqlite.pragma('journal_mode = WAL');
   sqlite.pragma('busy_timeout = 5000');

   export const db = drizzle(sqlite, { schema });
   ```

3. Create `cli/ui/Table.tsx` — a reusable Ink table component. Props: `headers: string[]`, `rows: string[][]`, `widths: number[]`. Renders a header row in bold, then data rows, using `<Box>` with `width` prop for column alignment. Also export a helper `formatDate(unixMs: number): string` that formats a Unix millisecond timestamp as a readable date string (e.g., "2026-02-23 10:30"). Use Ink's `<Box flexDirection="column">` for rows and `<Box>` for columns. Each cell is a `<Text>` inside a `<Box width={w}>`.

4. Update `tsconfig.json`: Add `"jsx": "react-jsx"` to compilerOptions so tsx can compile JSX in cli/ files. Also add `"cli/**/*"` to the `include` array alongside `"src/**/*"`.

1. Create `cli/ui/StatusView.tsx` — an Ink component that accepts props `{ pm2Status: string; uptime: number | null; memory: number | null; contactCount: number; groupCount: number; draftCount: number }` and renders a formatted status display. Show:
   - Bot status with color (green for "online", red for "stopped"/"errored", yellow for anything else)
   - Uptime formatted as "Xd Xh Xm" (or "N/A" if null)
   - Memory usage in MB (or "N/A" if null)
   - Active contacts count (mode != 'off')
   - Tracked groups count (active = true)
   - Pending drafts count
   Use `<Box flexDirection="column">` with labeled rows.

2. Create `cli/commands/status.ts`:
   - Export `addStatusCommand(program: Command): void`
   - Register `program.command('status').description('Show bot status, uptime, and counts')`
   - In the async action handler:
     a. Query DB synchronously for counts:
        - Active contacts: `db.select({ count: sql<number>\`count(*)\` }).from(contacts).where(sql\`mode != 'off'\`).get()`
        - Active groups: `db.select({ count: sql<number>\`count(*)\` }).from(groups).where(eq(groups.active, true)).get()`
        - Pending drafts: `db.select({ count: sql<number>\`count(*)\` }).from(drafts).where(eq(drafts.status, 'pending')).get()`
     b. Query PM2 async:
        - Create a `getBotStatus()` promise function that calls `pm2.connect(true, cb)` then `pm2.describe('whatsapp-bot', cb)`, extracts `pm2_env.status`, uptime (`Date.now() - pm2_env.pm_uptime`), and `monit.memory`, always calls `pm2.disconnect()` before resolving. On error, resolve with `{ status: 'pm2_unavailable', uptime: null, memory: null }`.
     c. Render the `StatusView` component with `render()` + `waitUntilExit()`. Pass all data as props. Call `exit()` from `useApp()` after the PM2 query resolves.
   - Import `db` from `../db.js`, schema tables from `../../src/db/schema.js`, pm2 from `pm2`.
   - For PM2 types: pm2's `describe` callback returns `ProcessDescription[]`. Access `proc.pm2_env?.status` and `proc.pm2_env?.pm_uptime` and `proc.monit?.memory`. You may need to cast or use optional chaining since PM2 types can be loose.

3. Create `cli/bot.ts`:
   - First line: `#!/usr/bin/env tsx`
   - Import `Command` from `commander`
   - Import `addStatusCommand` from `./commands/status.js`
   - Import placeholder registration functions for future commands (just import stubs that will be replaced in plans 07-02 and 07-03): Do NOT create stubs. Only import and register `addStatusCommand` for now. Plans 07-02 and 07-03 will add their own imports.
   - Create root program: `new Command().name('bot').description('WhatsApp bot management CLI').version('1.0.0')`
   - Call `addStatusCommand(program)`
   - Call `await program.parseAsync(process.argv)`
   - Make the file executable: `chmod +x cli/bot.ts`

4. Add to `package.json`:
