<research_facts>
Resolved before planning via grep + file reads:

1. **pm-authority already exposes pick-lesson + pick-variant + lesson-runs endpoints** (Phase 33, plans 33-03/33-04). Contract: POST /v1/posts/{id}/pick-lesson with body `{candidate_id: int}` → 202 JobAccepted (async slow path). POST /v1/posts/{id}/pick-variant with body `{variant_id: int}` → mixed 200 PostDTO (fast: hook variants or already-has-image) or 202 JobAccepted (slow: lesson variants needing fal.ai image gen).

2. **whatsapp-bot already proxies all three** (Phase 34-03). Routes live in `src/api/linkedin/routes/writes.ts` at:
    - POST /api/linkedin/posts/:id/pick-lesson (line ~230)
    - POST /api/linkedin/posts/:id/pick-variant (line ~181, mixed 200/202 pass-through)
    - POST /api/linkedin/lesson-runs (line ~280+)
   Request schemas `PickLessonRequestSchema` + `PickVariantRequestSchema` exist in `src/api/linkedin/schemas.ts` (lines 174-186). **Zero new proxy routes are needed for Phase 37.**

3. **Existing PostDTO fields** (`pm-authority/services/http/schemas.py`): id, sequence_id, position, status, perspective, language, content, content_he, image, variants, lesson_candidates, regeneration_count, regeneration_capped, share_urn, scheduled_at, published_at, created_at, updated_at, analytics. **Missing for Phase 37's locked page-header requirement:** project_name, source_snippet. Must be added here.

4. **sequences table schema** (`pm-authority/review/manager.py` lines 34-42): `CREATE TABLE IF NOT EXISTS sequences (id TEXT PRIMARY KEY, project_name TEXT NOT NULL, created_at TEXT NOT NULL, perspective TEXT, language TEXT, context_json TEXT, mode TEXT NOT NULL DEFAULT 'sequence')`. `project_name` is required; `context_json` is nullable (some early rows pre-dating the column backfill may be NULL) → must be treated as Optional in the DTO and sliced to `[:500]` for the snippet OR mapped to None.

5. **post_variants table schema** (lines 59-67): has `created_at TEXT NOT NULL DEFAULT (datetime('now'))` column already. Currently NOT surfaced by dto_mapper._variant_rows_to_dtos (line 62+). Same for `lesson_candidates` (lines 68-75, `created_at TEXT NOT NULL DEFAULT (datetime('now'))`).

6. **No model_used / token_cost column exists anywhere in pm-authority** (confirmed via `grep -rn 'token_cost\|model_used\|total_tokens' pm-authority --include='*.py'`). CONTEXT §decisions lists "generation metadata (model, timestamp, token cost)" but only `created_at` (= timestamp) is available today. The GenerationMetadata component will render:
    - `created_at` as a relative string ("generated 4m ago")
    - a hard-coded "Claude" label (pm-authority's generator is always Claude CLI per generation/generator.py)
    - token_cost OMITTED (rendered as a subtle "—" tail OR entirely skipped) to keep the contract pragmatic. Documented assumption in this plan.

7. **Dashboard router** (`dashboard/src/router.tsx` lines 16-39): createBrowserRouter with a children array under AppLayout. Existing route `{ path: 'linkedin/queue', element: <LinkedInQueue /> }`. Adding two siblings `{ path: 'linkedin/queue/posts/:id/lesson', ... }` and `{ path: 'linkedin/queue/posts/:id/variant', ... }` — NO new layout wrappers needed; AppLayout already handles the chrome.

8. **DashboardPostSchema's variants/lesson_candidates are typed `z.array(z.any())`** (`dashboard/src/api/linkedinSchemas.ts` lines 63-64 + comment lines 44-49). The existing comment explicitly says "The UI only reads .length off them — validating their inner shape would couple the dashboard to the full variant/lesson contracts unnecessarily." Phase 37 is the point where that tradeoff flips: the lesson + variant pages ARE the inner-shape consumers. We upgrade to strongly-typed schemas here.

9. **existing Post Schema passthrough** (`.passthrough()` at line 76) guarantees the Zod parse does not reject unknown fields — adding project_name + source_snippet on the server side BEFORE this plan lands the dashboard-side schema update is safe and the old dashboard code will simply ignore the new fields.

10. **GenerationMetadata rendering format (locked by this plan):** `<span>Generated {relative} · Claude</span>` — single-line, 12px, muted-foreground, no badge. Reused verbatim on both lesson cards and variant cards.

11. **StickyConfirmBar rendering format (locked by this plan):** bottom-sticky `<div className="sticky bottom-0 ...">` wrapping a right-aligned primary button. Accepts `{ label: string, disabled: boolean, onConfirm: () => void, helper?: ReactNode }`. Position: below the content scroll area, above the viewport bottom edge with `safe-area-inset-bottom` padding on mobile.

12. **Stub page rendering for Wave 2 hand-off:** each stub page receives `{ id }` from `useParams<{id: string}>()` and renders `<div>Loading lesson for {id}…</div>`. Wave 2 replaces the page body; the file path + default export stay stable so router.tsx imports don't need to change.
</research_facts>
