<task type="auto">
  <name>Task 1: Implement Baileys connection factory and reconnect logic</name>
  <files>
    src/whatsapp/connection.ts
    src/whatsapp/reconnect.ts
  </files>
  <action>
    1. Create `src/whatsapp/connection.ts`:
       - Import `makeWASocket`, `useMultiFileAuthState`, `Browsers` from '@whiskeysockets/baileys'.
       - Import pino for logger (set level to 'silent' to suppress Baileys internal noise).
       - Import config for AUTH_DIR path.
       - Export async function `createSocket()`:
         a. Call `useMultiFileAuthState(config.AUTH_DIR)` to get `{ state, saveCreds }`.
         b. Create socket with `makeWASocket({ auth: state, logger, browser: Browsers.ubuntu('Chrome'), markOnlineOnConnect: false, getMessage: async () => undefined })`.
         c. Wire `sock.ev.on('creds.update', saveCreds)` — this persists session after QR scan.
         d. Return `{ sock, saveCreds }`.
       - IMPORTANT: Do NOT add reconnect logic here. The reconnect handler creates a NEW socket (Baileys sockets cannot be reused after close).
       - IMPORTANT: All imports use .js extensions for ESM.

    2. Create `src/whatsapp/reconnect.ts`:
       - Import `DisconnectReason` from '@whiskeysockets/baileys'.
       - Import `Boom` from '@hapi/boom' (bundled with Baileys, no extra install needed).
       - Export a class or module that tracks retry state:
         - `retryCount`: starts at 0, increments on each reconnect attempt
         - `MAX_RETRIES`: 10
         - `resetRetries()`: called when connection opens successfully
       - Export function `handleConnectionUpdate(update, callbacks)` where callbacks has:
         - `onQR: (qr: string) => void` — log QR to terminal
         - `onOpen: () => void` — log success, reset retry count
         - `onReconnect: (delayMs: number) => void` — schedule reconnect
         - `onLoggedOut: () => void` — delete session, do not reconnect
         - `onMaxRetriesReached: () => void` — log fatal, let PM2 handle restart
       - Logic:
         - If `qr` present in update: call `onQR(qr)` (log QR code as text for terminal scanning using the qr-terminal approach -- use `qrcode-terminal` package or just log the raw QR string; the user will scan from their phone's WhatsApp app).
         - If `connection === 'close'`:
           - Extract statusCode from `(lastDisconnect?.error as Boom)?.output?.statusCode`
           - If 401 (loggedOut), 403 (forbidden), or 500 (badSession): call `onLoggedOut()`, return
           - If 440 (connectionReplaced): log warning, do NOT reconnect, return
           - Otherwise: compute delay as `Math.min(1000 * 2 ** retryCount, 60_000)`, increment retryCount, check against MAX_RETRIES, call `onReconnect(delay)` or `onMaxRetriesReached()`
         - If `connection === 'open'`: call `onOpen()`, reset retryCount to 0
       - NOTE: Install `qrcode-terminal` for QR display: `npm install qrcode-terminal` and `npm install -D @types/qrcode-terminal` (check if types exist, if not use `// @ts-ignore` or declare module).
  </action>
  <verify>
    1. `npx tsc --noEmit` — zero TypeScript errors.
    2. Manually inspect that reconnect.ts handles all DisconnectReason codes from the research doc.
    3. Verify that connection.ts does NOT attempt to reuse a closed socket.
  </verify>
  <done>
    connection.ts exports createSocket() that produces an authenticated Baileys socket. reconnect.ts exports handleConnectionUpdate() that routes disconnect reasons correctly: backoff for transient errors, session delete for auth failures, no reconnect for connectionReplaced.
  </done>

<task type="auto">
  <name>Task 2: Wire connection and reconnect into main entry point</name>
  <files>

  <action>
    1. Update `src/index.ts` to wire the Baileys connection:
       - Import `createSocket` from './whatsapp/connection.js'.
       - Import `handleConnectionUpdate` (and retry state) from './whatsapp/reconnect.js'.
       - Import `rimraf` or `fs.rm` for session deletion.
       - In `main()`, after database init:
         a. Define `async function startSocket()`:
            - Call `const { sock, saveCreds } = await createSocket()`
            - Wire `sock.ev.on('connection.update', (update) => handleConnectionUpdate(update, { ... }))`
            - The `onQR` callback: use qrcode-terminal to render QR in terminal (import `qrcode` from 'qrcode-terminal' and call `qrcode.generate(qr, { small: true })`)
            - The `onOpen` callback: log "Connected to WhatsApp" with pino logger
            - The `onReconnect` callback: `setTimeout(() => startSocket(), delayMs)` — creates a FRESH socket each time (critical: never reuse closed Baileys socket)
            - The `onLoggedOut` callback: delete AUTH_DIR contents (`fs.rm(config.AUTH_DIR, { recursive: true, force: true })`), then log "Session expired — restart to re-scan QR" and `process.exit(1)` (PM2 will restart, user rescans)
            - The `onMaxRetriesReached` callback: log fatal error and `process.exit(1)` (PM2 will restart with its own delay)
         b. Call `await startSocket()`
       - Handle graceful shutdown: `process.on('SIGTERM', ...)` and `process.on('SIGINT', ...)` — log "Shutting down..." and `process.exit(0)`.
       - IMPORTANT: The startSocket function must be callable recursively for reconnects (each call creates a new socket).
  </action>

    3. After scanning QR: should print "Connected to WhatsApp" and files should appear in data/auth/.
    4. Kill and restart `npx tsx src/index.ts` — should connect WITHOUT showing QR (session persisted).
