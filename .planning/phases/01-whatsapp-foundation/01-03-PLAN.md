<task type="auto">
  <name>Task 1: Create message database queries and receive pipeline</name>
  <files>
    src/db/queries/messages.ts
    src/pipeline/messageHandler.ts
  </files>
  <action>
    1. Create `src/db/queries/messages.ts`:
       - Import `db` from '../client.js' and `messages` table from '../schema.js'.
       - Import `eq` from 'drizzle-orm'.
       - Export `async function insertMessage(msg: { id: string; contactJid: string; fromMe: boolean; body: string; timestamp: number })`:
         - Use `db.insert(messages).values({ ...msg }).onConflictDoNothing()` — this is the dedup mechanism. The message `id` is the primary key, so duplicate inserts are silently ignored.
         - Return the result (or void — the important thing is no error on duplicate).
       - Export `async function messageExists(id: string): Promise<boolean>`:
         - Query `db.select({ id: messages.id }).from(messages).where(eq(messages.id, id)).limit(1)`
         - Return `result.length > 0`
       - Export `async function getRecentMessages(contactJid: string, limit = 20)`:
         - Select from messages where contactJid matches, order by timestamp desc, limit.
         - This will be used by Phase 2 for context but define it now.

    2. Create `src/pipeline/messageHandler.ts`:
       - Import `insertMessage` from '../db/queries/messages.js'.
       - Import types from '@whiskeysockets/baileys' (`WAMessage`, `MessageUpsertType` or equivalent types).
       - Import pino logger.
       - Export `async function handleMessagesUpsert(upsert: { messages: WAMessage[]; type: MessageUpsertType })`:
         a. If `upsert.type !== 'notify'`, return immediately. (Type 'append' is history sync, not real-time.)
         b. For each `msg` in `upsert.messages`:
            - Skip if `msg.key.fromMe` is true (we only process incoming).
            - Extract `jid = msg.key.remoteJid`. Skip if falsy.
            - Skip if `jid.endsWith('@g.us')` (group messages — out of scope per PROJECT.md).
            - Skip if `jid === 'status@broadcast'` (WhatsApp status updates).
            - Extract `messageId = msg.key.id`. Skip if falsy.
            - Extract text body: check `msg.message?.conversation` || `msg.message?.extendedTextMessage?.text` || ''. If body is empty string, skip (non-text message like image/sticker).
            - Extract timestamp: `(msg.messageTimestamp as number) * 1000` (Baileys gives seconds, we store ms).
            - Call `await insertMessage({ id: messageId, contactJid: jid, fromMe: false, body, timestamp })`.
            - Log at debug level: `"Stored message ${messageId} from ${jid}"`.
         c. Wrap the entire loop in try/catch — log errors but don't crash. Individual message failures should not prevent processing subsequent messages (use per-message try/catch inside the loop).
       - IMPORTANT: The dedup is handled by the primary key constraint + onConflictDoNothing in insertMessage. No need for an in-memory Set or pre-check query. This handles rapid redelivery on reconnect.
  </action>
  <verify>
    1. `npx tsc --noEmit` — zero errors.
    2. Write a quick inline test: create a temporary script that calls `insertMessage` twice with the same ID and verifies no error and only one row exists. Run with tsx.
  </verify>
  <done>
    messageHandler filters out group messages, status broadcasts, non-text messages, and fromMe messages. Incoming text messages are persisted to SQLite. Duplicate message IDs are silently ignored via onConflictDoNothing. Per-message error handling prevents cascading failures.
  </done>
</task>

  <name>Task 2: Create send function and wire everything into index.ts</name>
  <files>

  <action>
    1. Create `src/whatsapp/sender.ts`:
       - Import WASocket type from '@whiskeysockets/baileys'.
       - Import pino logger.
       - Export `async function sendWithDelay(sock: WASocket, jid: string, text: string, options?: { minDelay?: number; maxDelay?: number })`:
         a. Destructure `{ minDelay = 1500, maxDelay = 4000 } = options ?? {}`.
         b. Call `await sock.sendPresenceUpdate('composing', jid)` — shows "typing..." to recipient.
         c. Compute `delay = minDelay + Math.random() * (maxDelay - minDelay)`.
         d. `await new Promise(resolve => setTimeout(resolve, delay))`.
         e. Call `await sock.sendPresenceUpdate('paused', jid)` — stop typing indicator.
         f. Call `const sent = await sock.sendMessage(jid, { text })`.
         g. Log at info level: sent message to jid with delay rounded to ms.
         h. Return `sent` (the sent message object, for caller to store if needed).
       - Export a helper `function extractTextBody(msg: WAMessage): string`:
         - Return `msg.message?.conversation || msg.message?.extendedTextMessage?.text || ''`.
         - This is shared logic also used by messageHandler — if messageHandler already has it inline, refactor to import from here.

    2. Update `src/index.ts`:
       - Import `handleMessagesUpsert` from './pipeline/messageHandler.js'.
       - Inside `startSocket()`, after creating the socket and wiring connection.update:
         a. Wire `sock.ev.on('messages.upsert', handleMessagesUpsert)`.
         b. Log "Message pipeline active" after wiring.
       - The send function is NOT automatically called on incoming messages (that's Phase 2's AI response logic). It is available as an export for manual testing or future use.
       - Add a comment in index.ts: `// Phase 2 will wire AI response generation -> sendWithDelay for draft/auto mode`.
       - Refactor `extractTextBody` if duplicated: move to sender.ts and import in messageHandler.ts.
