<task type="auto">
  <name>Task 1: Initialize TypeScript ESM project with all dependencies</name>
  <files>
    package.json
    tsconfig.json
    .gitignore
    .env.example
    .env
  </files>
  <action>
    1. Initialize git repo: `git init`
    2. Run `npm init -y` and set `"type": "module"` and `"engines": { "node": ">=20" }` in package.json.
    3. Install production dependencies:
       ```
       npm install @whiskeysockets/baileys drizzle-orm better-sqlite3 pino pino-pretty dotenv zod
       ```
    4. Install dev dependencies:
       ```
       npm install -D typescript tsx @tsconfig/node20 @types/node @types/better-sqlite3 drizzle-kit
       ```
    5. Create tsconfig.json extending @tsconfig/node20 with:
       - `"module": "ESNext"`, `"moduleResolution": "bundler"`
       - `"outDir": "./dist"`, `"rootDir": "./src"`
       - `"strict": true`, `"skipLibCheck": true`
       - `"resolveJsonModule": true`, `"esModuleInterop": true`
    6. Create .gitignore with: node_modules/, dist/, data/, .env, logs/, *.db
    7. Create .env.example with placeholder values:
       ```
       NODE_ENV=development
       LOG_LEVEL=info
       AUTH_DIR=./data/auth
       DB_PATH=./data/bot.db
       ```
    8. Create .env copying .env.example with same values for local dev.
    9. Create empty directories: `mkdir -p src/db/queries src/whatsapp src/pipeline data/auth logs drizzle`
    10. Add scripts to package.json:
        - `"dev": "tsx watch src/index.ts"`
        - `"start": "tsx src/index.ts"`
        - `"db:generate": "drizzle-kit generate"`
        - `"db:migrate": "drizzle-kit migrate"`
  </action>
  <verify>
    Run `npx tsc --noEmit` — should succeed with zero errors (after creating minimal src/index.ts in Task 2).
    Run `node -e "import('./package.json', { assert: { type: 'json' } }).then(m => console.log(m.default.type))"` — should print "module".
  </verify>
  <done>
    package.json has type=module with all dependencies installed, tsconfig.json compiles without errors, .gitignore excludes data/secrets, directory structure exists.
  </done>
</task>

<task type="auto">
  <name>Task 2: Create database schema, client, config, PM2 config, and entry point</name>
  <files>
    src/config.ts
    src/db/schema.ts
    src/db/client.ts
    src/index.ts
    drizzle.config.ts
    ecosystem.config.cjs
  </files>
  <action>
    1. Create `src/config.ts`:
       - Use zod to define and parse environment schema: NODE_ENV (default 'development'), LOG_LEVEL (default 'info'), AUTH_DIR (default './data/auth'), DB_PATH (default './data/bot.db').
       - Load dotenv at top of file.
       - Export a typed `config` object (parsed result).
       - Exit process with clear error message if validation fails.

    2. Create `src/db/schema.ts`:
       - Define `messages` table using `sqliteTable`:
         - `id` text primary key (Baileys message key.id)
         - `contactJid` text not null
         - `fromMe` integer (boolean mode) not null
         - `body` text not null default ''
         - `timestamp` integer not null (epoch ms)
         - `processed` integer (boolean mode) default false
         - `createdAt` integer not null with $defaultFn(() => Date.now())
       - Add composite index on (contactJid, timestamp) named 'idx_messages_contact_ts'.
       - Define `contacts` table (for future use, keep minimal):
         - `jid` text primary key
         - `name` text
         - `mode` text not null default 'off' (values: 'off', 'draft', 'auto')
         - `createdAt` integer not null with $defaultFn(() => Date.now())
         - `updatedAt` integer not null with $defaultFn(() => Date.now())

    3. Create `src/db/client.ts`:
       - Import Database from better-sqlite3, drizzle from drizzle-orm/better-sqlite3.
       - Import config for DB_PATH.
       - Create better-sqlite3 instance with config.DB_PATH.
       - Enable WAL mode: `sqlite.pragma('journal_mode = WAL')`.
       - Export `db` as drizzle(sqlite, { schema }).
       - Export an `initDb()` function that runs drizzle-kit migrate programmatically OR uses `db.run(sql)` to create tables if they don't exist. Use the push approach: import `migrate` from 'drizzle-orm/better-sqlite3/migrator' and run `migrate(db, { migrationsFolder: './drizzle' })`. If no migrations exist yet, generate them first during setup.

    4. Create `drizzle.config.ts`:
       - Export default config: schema path 'src/db/schema.ts', out './drizzle', dialect 'sqlite', dbCredentials with url from process.env.DB_PATH or './data/bot.db'.

    5. Create `ecosystem.config.cjs` (MUST be .cjs for ESM project):
       ```javascript
       module.exports = {
         apps: [{
           name: 'whatsapp-bot',
           script: './src/index.ts',
           interpreter: 'node',
           interpreterArgs: '--import tsx',
           watch: false,
           autorestart: true,
           max_restarts: 10,
           restart_delay: 5000,
           env: { NODE_ENV: 'production' },
           log_date_format: 'YYYY-MM-DD HH:mm:ss',
           error_file: './logs/bot-error.log',
           out_file: './logs/bot-out.log',
         }],
       };
       ```

    6. Create `src/index.ts`:
       - Import config from './config.js' (note .js extension for ESM).
       - Import db initialization from './db/client.js'.
       - Create a pino logger with config.LOG_LEVEL.
       - In an async main() function:
         a. Log "Starting WhatsApp Bot..."
         b. Initialize database (run migrations / ensure tables exist)
         c. Log "Database initialized"
         d. Log "WhatsApp Bot started (scaffold only — connection in next plan)"
       - Call main().catch(err => { logger.fatal(err); process.exit(1); })
       - IMPORTANT: All local imports must use .js extensions (ESM requirement).
  </action>
  <verify>
    1. Run `npx drizzle-kit generate` — should produce migration SQL in drizzle/ folder.
    2. Run `npx tsx src/index.ts` — should print startup logs and create data/bot.db.
    3. Run `sqlite3 data/bot.db ".tables"` — should show 'messages' and 'contacts' tables.
    4. Run `npx tsc --noEmit` — zero TypeScript errors.
  </verify>
  <done>
    Running `npx tsx src/index.ts` produces "Database initialized" log, creates data/bot.db with messages and contacts tables, and exits cleanly. PM2 config exists at ecosystem.config.cjs. All TypeScript compiles without errors.
  </done>
</task>
