    Declare two Zod schemas and their inferred types. The shape MUST match the server row (`Actionable` in `src/db/queries/actionables.ts` line 16 → inferred from `src/db/schema.ts` line 253 actionables table).

    ```ts
    import { z } from 'zod';

    export const ActionableSchema = z
      .object({
        id: z.string(),
        sourceType: z.enum(['commitment', 'task', 'user_command']),
        sourceContactJid: z.string(),
        sourceContactName: z.string().nullable(),
        sourceMessageId: z.string().nullable(),
        sourceMessageText: z.string(),
        detectedLanguage: z.enum(['he', 'en']),
        originalDetectedTask: z.string(),
        task: z.string(),
        status: z.enum(['pending_approval', 'approved', 'rejected', 'fired', 'expired']),
        detectedAt: z.number(),
        fireAt: z.number().nullable(),
        enrichedTitle: z.string().nullable(),
        enrichedNote: z.string().nullable(),
        todoTaskId: z.string().nullable(),
        todoListId: z.string().nullable(),
        approvalPreviewMessageId: z.string().nullable(),
        createdAt: z.number(),
        updatedAt: z.number(),
      })
      .passthrough(); // tolerate forward-compatible new columns from server

    export type Actionable = z.infer<typeof ActionableSchema>;

    export const ActionablesUpdatedPayloadSchema = z.object({
      pending: z.array(ActionableSchema),
      recent: z.array(ActionableSchema),
    });
    ```

    Use `.passthrough()` not `.strict()` — the server may add columns and the dashboard must not fail closed.

    **File B — `dashboard/src/hooks/useActionablesStream.ts`:**

    Model on `dashboard/src/hooks/useLinkedInQueueStream.ts` (read it end-to-end first). The shape:

    ```ts
    export type StreamStatus = 'connecting' | 'open' | 'reconnecting' | 'error';

    export function useActionablesStream(): {
      pending: Actionable[] | null;
      recent: Actionable[] | null;
      status: StreamStatus;
    }
    ```

    Logic (mirrors LinkedIn hook 1:1):
    1. `new EventSource(sseUrl('/api/actionables/stream'))` on mount.
    2. Listen for `actionables.updated` events. JSON.parse then `ActionablesUpdatedPayloadSchema.safeParse`. On success, setPending + setRecent. On JSON parse or Zod failure, log `console.error('[useActionablesStream] schema drift on actionables.updated; falling back to polling', ...)` and switch to 5s polling.
    3. Polling fallback (`fastPoll`): calls `apiFetch<{actionables: unknown[]}>('/api/actionables/pending')` + `apiFetch<{actionables: unknown[]}>('/api/actionables/recent')`, safeParses both arrays via `z.array(ActionableSchema)`, keeps last-known-good on drift.
    4. `es.onopen` → setStatus('open'). `es.onerror` → setStatus('reconnecting').
    5. Cleanup: `es.close()` + clear the fallback interval.

    Import `apiFetch`, `sseUrl` from `@/api/client`. Import schemas from `@/api/actionablesSchemas`. Do NOT add React Query — the existing pattern is plain `useState` + `useEffect` (CONTEXT §4 equivalent for linkedin hook).

    Return `{ pending: null, recent: null, status: 'connecting' }` on initial render so the page can render a skeleton.
