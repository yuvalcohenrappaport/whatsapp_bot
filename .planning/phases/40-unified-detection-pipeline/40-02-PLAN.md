Add a small helper near the top of the file (after imports, before `createMessageHandler`):

```typescript
import { processDetection } from '../actionables/detectionService.js';
import { getSetting } from '../db/queries/settings.js';

function routeDetection(params: Parameters<typeof processCommitment>[0]): void {
  const mode = getSetting('v1_8_detection_pipeline') ?? 'dark_launch';
  if (mode === 'legacy') {
    processCommitment(params).catch((err) =>
      logger.error({ err, messageId: params.messageId }, 'legacy processCommitment failed'),
    );
  } else {
    // default: 'dark_launch' (v1.8 silent queue)
    processDetection(params).catch((err) =>
      logger.error({ err, messageId: params.messageId }, 'processDetection failed'),
    );
  }
}
```

Both `processCommitment` and `processDetection` take the same param shape (messageId, contactJid, contactName, text, timestamp, fromMe). If they diverge for some reason, adapt the adapter — but the plan is that they stay type-compatible (see Plan 40-01 Task 2).
